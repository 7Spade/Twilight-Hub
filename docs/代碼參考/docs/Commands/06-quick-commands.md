## 💡 快速指令最佳實踐

### 使用建議
1. **明確目標**: 確保指令目標明確
2. **包含範圍**: 指定操作的範圍
3. **考慮影響**: 評估操作的影響
4. **準備回滾**: 為重要操作準備回滾方案

### 組合使用
```
# 創建並測試
創建用戶管理功能並運行測試
創建文件上傳功能並檢查性能
創建 API 接口並驗證安全性

# 修復並優化
修復所有錯誤並優化性能
修復所有警告並清理代碼
修復所有問題並更新文檔

# 檢查並報告
檢查項目狀態並生成報告
檢查代碼質量並生成建議
檢查性能指標並生成分析
```

### 避免的快速指令模式
```
❌ "修復一切" - 太模糊
❌ "優化所有" - 沒有具體目標
❌ "清理項目" - 缺少清理策略

✅ "// TODO: [P0] FIX 修復所有緊急錯誤"
✅ "// TODO: [P1] PERF 優化所有組件的渲染性能"
✅ "// TODO: [P2] REFACTOR 清理所有未使用的導入和變數"
```
